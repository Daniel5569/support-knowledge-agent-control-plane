# Publishing Guide

Use this guide before making the repository public.

1. Install dependencies with a working Node/npm installation.
2. Run `npm run check`.
3. Review `git status --short` and make sure only intended source/docs/config files are staged.
4. Confirm no `.env`, logs, databases, screenshots, caches, `node_modules`, `.next`, or local-only artifacts are staged.
5. Create the GitHub repository.
6. Push to GitHub and confirm the rendered README looks correct.

The Italian handoff guide in `docs/PUBLISHING_GUIDE_IT.md` has the fuller checklist and project positioning notes.
